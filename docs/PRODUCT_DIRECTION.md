# Product Direction

Mo Actions is intended to be a Rails engine for defining, running, and observing operational actions from an internal dashboard.

Likely future capabilities include:

- definable actions
- a dashboard for operators
- arguments and generated forms
- execution records
- background execution
- progress reporting
- logs
- batching
- authorization
- preflight or confirmation flows
- auditability
- retention and cleanup

These are directional goals, not a phase plan. Add capabilities only when the next working slice needs them.
